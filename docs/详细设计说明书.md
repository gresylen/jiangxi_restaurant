## 一、引言

### 1.1 文档目的
本文档基于需求规格说明书和概要设计说明书，详细描述江西小炒饭店点菜系统的技术实现细节，包括系统架构、模块实现、数据结构、接口设计、界面设计等，为开发人员提供具体的实现指导。

### 1.2 术语定义
| 术语 | 解释 |
|------|------|
| SPA | 单页面应用（Single Page Application），指整个应用只有一个HTML页面，通过JavaScript动态更新内容 |
| AJAX | 异步JavaScript和XML（Asynchronous JavaScript and XML），用于在不刷新页面的情况下与服务器交换数据 |
| JSON | JavaScript对象表示法（JavaScript Object Notation），一种轻量级的数据交换格式 |
| Flask | Python的一个轻量级Web框架 |
| CORS | 跨域资源共享（Cross-Origin Resource Sharing），允许不同域的客户端访问服务器资源 |

## 二、系统架构详细设计

### 2.1 系统整体架构
- **架构模式**：客户端-服务器架构，采用单页面应用（SPA）模式
- **服务器**：Python 3.x + Flask框架，提供RESTful API接口
- **客户端**：HTML5 + CSS3 + JavaScript + jQuery，实现单页面应用
- **数据存储**：JSON文件存储核心数据，图片文件存储菜品图片
- **离线处理**：使用localStorage存储临时数据，网络恢复后自动同步

### 2.2 技术栈详细说明
- **后端**：
  - Python 3.8+
  - Flask 2.0+
  - Flask-CORS 3.0+（处理跨域请求）
  - os、json、datetime等标准库（用于文件操作和数据处理）
- **前端**：
  - HTML5
  - CSS3（使用Flexbox和Grid布局）
  - JavaScript（ES6+）
  - jQuery 3.6+（简化DOM操作和AJAX请求）
- **数据存储**：
  - JSON文件（存储订单、菜品、价格历史、统计数据）
  - 图片文件（存储菜品图片）
  - localStorage（存储离线数据）

## 三、模块详细设计

### 3.1 点菜模块
- **功能**：菜品展示、添加、修改、删除
- **实现细节**：
  - **菜品展示**：
    - 从服务器获取菜品列表（GET /api/dishes）
    - 以网格形式展示菜品图片、名称和价格
    - 支持按类别筛选（如果实现类别功能）
  - **添加菜品**：
    - 点击菜品图片添加到当前订单
    - 支持手动输入临时菜品（无图片）
    - 添加时默认份数为1，可修改
  - **修改菜品**：
    - 修改菜品份数（通过输入框）
    - 添加备注（通过文本输入框）
  - **删除菜品**：
    - 点击已点菜品旁的删除按钮
    - 弹出确认对话框
    - 确认后删除菜品

### 3.2 后厨模块
- **功能**：菜品状态管理、实时更新
- **实现细节**：
  - **实时更新**：
    - 使用AJAX轮询（每3秒）获取最新订单状态
    - 比较本地数据与服务器数据，只更新变化部分
  - **状态管理**：
    - 待做菜品列表：显示所有未完成的菜品
    - 已完成菜品列表：显示已完成的菜品
    - 点击"完成"按钮标记菜品为已完成
    - 点击"重做"按钮将已完成菜品重新标记为待做
  - **排序规则**：
    - 按点菜时间排序
    - 相同菜品排在一起，显示总份数

### 3.3 结账模块
- **功能**：账单生成、支付记录、发票管理
- **实现细节**：
  - **账单生成**：
    - 选择桌号后，从服务器获取该桌订单数据
    - 计算菜品小计和总计
    - 显示详细账单（菜品名称、单价、数量、小计）
  - **支付记录**：
    - 输入实际支付金额
    - 记录支付时间
  - **发票管理**：
    - 结账时可输入发票信息（公司名称、税号）
    - 保存发票信息到订单数据

### 3.4 管理模块
- **功能**：菜品管理、价格调整、数据备份、统计分析
- **实现细节**：
  - **菜品管理**：
    - 添加新菜品：上传图片、输入菜名和默认价格
    - 图片上传：使用FormData对象上传图片文件
    - 保存菜品信息到服务器
  - **价格调整**：
    - 修改单个菜品价格：输入新价格，点击保存
    - 批量调整价格：输入调整比例，点击批量修改
    - 记录价格修改历史
  - **数据备份**：
    - 手动备份：点击"备份"按钮，生成备份文件
    - 自动备份：每天营业结束后自动执行
  - **统计分析**：
    - 热销菜品：统计一段时间内销量最高的菜品
    - 每日营业额：统计每天的总销售额
    - 使用图表展示统计数据（可选）

### 3.5 数据模块
- **功能**：数据存储、读取、备份、恢复
- **实现细节**：
  - **数据存储**：
    - 使用JSON文件存储数据
    - 每个数据类型对应一个JSON文件
    - 写入数据时使用原子操作，确保数据一致性
  - **数据读取**：
    - 读取JSON文件，解析为Python对象
    - 缓存数据到内存，提高读取速度
  - **数据备份**：
    - 复制所有数据文件到备份目录
    - 按日期命名备份文件夹
  - **数据恢复**：
    - 从备份目录复制数据文件到主目录
    - 恢复后重启服务

### 3.6 网络模块
- **功能**：数据同步、离线处理
- **实现细节**：
  - **数据同步**：
    - 客户端定期向服务器发送同步请求
    - 服务器返回最新数据
    - 客户端更新本地数据
  - **离线处理**：
    - 网络中断时，使用localStorage存储临时数据
    - 网络恢复后，自动将localStorage中的数据同步到服务器
    - 处理冲突：以服务器数据为准，或提示用户选择

## 四、数据结构详细设计

### 4.1 订单数据结构
```json
{
  "order_id": "20260421123456",  // 订单ID，格式：日期+时间戳
  "table_id": 1,  // 桌号，1-7
  "status": "未结账",  // 订单状态：未结账/已结账
  "create_time": "2026-04-21 12:34:56",  // 创建时间
  "items": [  // 菜品列表
    {
      "item_id": "1",  // 菜品项ID
      "dish_id": 1,  // 菜品ID（图片库菜品），手动输入时为null
      "dish_name": "红烧肉",  // 菜品名称
      "price": 38.0,  // 单价
      "quantity": 1,  // 数量
      "remark": "加辣",  // 备注
      "status": "待做",  // 菜品状态：待做/已完成
      "image_path": "images/dish1.jpg"  // 图片路径，手动输入时为null
    }
  ],
  "total_amount": 38.0,  // 总金额
  "actual_payment": 40.0,  // 实际支付金额
  "payment_time": "2026-04-21 13:00:00",  // 支付时间
  "invoice_info": "公司名称：XXX有限公司，税号：1234567890"  // 发票信息
}
```

## 五、接口详细设计

### 5.1 API接口详细设计

| 接口路径 | 方法 | 功能 | 请求参数 | 成功返回值（200 OK） | 失败返回值 |
|---------|------|------|---------|---------------------|-----------|
| `/api/dishes` | GET | 获取菜品列表 | 无 | `{"dishes": [{"dish_id": 1, "dish_name": "红烧肉", "price": 38.0, "image_path": "images/dish1.jpg"}]}` | `{"error": "获取菜品列表失败"}` |
| `/api/dishes` | POST | 添加新菜品 | form-data: dish_name (string), price (number), image (file) | `{"dish_id": 1}` | `{"error": "添加菜品失败"}` |
| `/api/dishes/{id}` | PUT | 修改菜品价格 | application/json: `{"price": 40.0}` | `{"success": true}` | `{"error": "修改价格失败"}` |
| `/api/orders` | GET | 获取订单列表 | query: table_id (optional) | `{"orders": [{"order_id": "20260421123456", "table_id": 1, "status": "未结账"}]}` | `{"error": "获取订单列表失败"}` |
| `/api/orders` | POST | 创建新订单 | application/json: `{"table_id": 1, "items": [{"dish_id": 1, "quantity": 1, "remark": "加辣"}]}` | `{"order_id": "20260421123456"}` | `{"error": "创建订单失败"}` |
| `/api/orders/{id}` | PUT | 更新订单 | application/json: `{"items": [{"item_id": "1", "quantity": 2}], "status": "已结账"}` | `{"success": true}` | `{"error": "更新订单失败"}` |
| `/api/orders/{id}` | DELETE | 删除订单 | 无 | `{"success": true}` | `{"error": "删除订单失败"}` |
| `/api/orders/{id}/items` | POST | 添加菜品到订单 | application/json: `{"dish_id": 2, "quantity": 1, "remark": "免葱"}` | `{"success": true}` | `{"error": "添加菜品失败"}` |
| `/api/orders/{id}/items/{item_id}` | DELETE | 从订单中删除菜品 | 无 | `{"success": true}` | `{"error": "删除菜品失败"}` |
| `/api/orders/{id}/status` | PUT | 更新订单状态 | application/json: `{"status": "已结账"}` | `{"success": true}` | `{"error": "更新状态失败"}` |
| `/api/orders/{id}/checkout` | POST | 结账 | application/json: `{"actual_payment": 40.0, "invoice_info": "公司名称：XXX有限公司"}` | `{"success": true}` | `{"error": "结账失败"}` |
| `/api/export` | GET | 导出订单 | query: date (optional) | TXT文件 | `{"error": "导出失败"}` |
| `/api/statistics` | GET | 获取统计数据 | query: date (optional) | `{"date": "2026-04-21", "total_orders": 10, "total_amount": 500.0, "hot_dishes": [...]}` | `{"error": "获取统计数据失败"}` |
| `/api/backup` | POST | 手动备份数据 | 无 | `{"success": true}` | `{"error": "备份失败"}` |

### 5.2 模块间接口详细设计
- **点菜模块 ↔ 数据模块**：
  - 调用方式：本地函数调用
  - 数据传递：通过参数传递菜品和订单数据
  - 示例：`data_module.save_order(order_data)`

- **后厨模块 ↔ 数据模块**：
  - 调用方式：本地函数调用
  - 数据传递：通过参数传递订单状态数据
  - 示例：`data_module.update_order_status(order_id, status)`

- **结账模块 ↔ 数据模块**：
  - 调用方式：本地函数调用
  - 数据传递：通过参数传递结账数据
  - 示例：`data_module.checkout_order(order_id, actual_payment, invoice_info)`

- **管理模块 ↔ 数据模块**：
  - 调用方式：本地函数调用
  - 数据传递：通过参数传递菜品、价格、备份等数据
  - 示例：`data_module.add_dish(dish_data)`

## 六、界面详细设计

### 6.1 点菜页
- **布局**：
  - 顶部：桌号选择区（1-7按钮，当前桌号高亮）
  - 主体：菜品图片网格（每行3个，响应式布局）
  - 底部：已点菜品列表、操作按钮区
- **元素**：
  - 菜品卡片：包含菜品图片、名称、价格，点击添加
  - 已点菜品项：包含菜品名称、数量、备注、删除按钮
  - 操作按钮：手动加菜、提交整桌、修改桌号
- **交互**：
  - 点击桌号按钮切换桌号
  - 点击菜品卡片添加到当前订单
  - 点击已点菜品的删除按钮删除菜品
  - 点击"手动加菜"按钮弹出弹窗，输入菜品信息
  - 点击"提交整桌"按钮保存订单

### 6.2 后厨页
- **布局**：
  - 顶部：页面标题"后厨管理"
  - 主体：左右两栏（待做菜品、已完成菜品）
  - 底部：无
- **元素**：
  - 待做菜品卡片：包含桌号、菜品图片、名称、备注、数量、"完成"按钮
  - 已完成菜品卡片：包含桌号、菜品图片、名称、备注、数量、"重做"按钮
- **交互**：
  - 点击"完成"按钮将菜品移至已完成列表
  - 点击"重做"按钮将菜品移回待做列表
  - 自动刷新（每3秒）

### 6.3 结账页
- **布局**：
  - 顶部：桌号选择区（1-7按钮）
  - 主体：订单明细表格
  - 底部：总计、实付金额输入框、"结账"按钮、"导出TXT"按钮
- **元素**：
  - 订单明细表格：包含菜品名称、单价、数量、小计
  - 总计：显示总金额
  - 实付金额输入框：输入客人实际支付金额
  - 发票信息输入框：输入发票信息（可选）
- **交互**：
  - 点击桌号按钮显示对应桌的订单明细
  - 输入实付金额
  - 输入发票信息（可选）
  - 点击"结账"按钮完成结账
  - 点击"导出TXT"按钮导出订单

### 6.4 管理页
- **布局**：
  - 顶部：页面标题"管理中心"
  - 主体：四个子页面（价格管理、添加菜品、数据备份、统计分析）
  - 底部：无
- **元素**：
  - 价格管理：菜品列表、价格修改输入框、"保存"按钮、"批量调整"按钮
  - 添加菜品：图片上传控件、菜名输入框、价格输入框、"保存"按钮
  - 数据备份："手动备份"按钮、备份历史列表
  - 统计分析：日期选择器、热销菜品图表、每日营业额图表
- **交互**：
  - 切换子页面
  - 修改菜品价格并保存
  - 上传图片并添加新菜品
  - 点击"手动备份"按钮备份数据
  - 查看统计图表

## 七、部署与集成详细设计

### 7.1 部署环境
- **服务器部署**：
  - 操作系统：Windows 10/11
  - Python版本：3.8+
  - 安装依赖：`pip install flask flask-cors`
  - 启动服务：`python app.py`（假设主文件为app.py）
  - 端口配置：默认5000
- **客户端配置**：
  - 设备：平板或手机（11英寸推荐）
  - 浏览器：Chrome、Safari等现代浏览器
  - 访问地址：`http://服务器IP:5000`
  - 添加到主屏幕：使用浏览器的"添加到主屏幕"功能

### 7.2 集成方案
- **系统与现有设备集成**：
  - 利用现有平板或手机作为客户端
  - 利用现有Windows笔记本作为服务器
  - 无需额外硬件投资
- **数据迁移方案**：
  - 系统初始化时创建默认菜品数据
  - 支持手动导入现有菜品数据（通过JSON文件）
  - 首次使用时引导用户添加基础菜品

### 7.3 系统维护
- **数据备份**：
  - 自动备份：每天23:59自动执行
  - 手动备份：管理页面提供手动备份功能
  - 备份路径：`backup/YYYYMMDD/`
- **系统更新**：
  - 代码更新：替换服务器上的代码文件
  - 重启服务：`Ctrl+C`停止服务，重新运行`python app.py`
- **故障处理**：
  - 网络故障：系统自动切换到离线模式
  - 数据丢失：从备份恢复数据
