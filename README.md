 江西小炒明档点菜与后厨协同系统
 面向“看菜点菜”模式小炒店的轻量级前后端协同解决方案

##  项目背景
江西小炒多为“明档点菜”（客人看冰柜食材点菜），每日食材不固定，且后厨厨师存在不识字的情况。传统纸质菜单易导致**漏菜、上菜不及时、沟通困难**等问题。
本系统针对此痛点，实现**点菜->后厨->结账**全流程数字化，通过图片辅助识别菜品，提升小店运营效率。
##  核心功能
-  **智能点菜**：支持图片库快速点菜 + 临时手动加菜（无图也可），自动记录桌号与备注。
- **后厨协同**：实时展示“待做/已完成”菜品，**数量用醒目的橙色大徽章显示**，支持完成/重做，一屏尽览。
-  **快速结账**：显示订单明细，支持实付金额修改与发票备注，一键导出当日营业报表（TXT）。
-  **灵活管理**：
  - **价格管理**：动态调整菜品价格。
  - **添加菜品**：支持上传菜品图片。
  - **统计分析**：查看今日营业额、热销菜品TOP6及近7日趋势图。
##  技术栈
| 层级 | 技术选型 |
| :--- | :--- |
| **后端框架** | Python 3.8+ / Flask |
| **前端框架** | Vue 2.6 (CDN) + 原生 JavaScript |
| **样式设计** | CSS3 (Flexbox/Grid，平板触摸优化) |
| **数据存储** | 本地 JSON 文件 (无需数据库，开箱即用) |
| **通信协议** | RESTful API + JSON |
## 快速启动（本地运行）
### 1. 克隆仓库
git clone https://github.com/gresylen/jiangxi-restaurant.git
cd jiangxi-restaurant
### 2. 安装依赖

确保已安装 Python 3.8+，然后安装 Flask：

bash

pip install flask

### 3. 启动服务器

直接运行 Python 文件：

bash

python app.py

### 4. 访问系统

- 本机访问：浏览器打开 `http://127.0.0.1:5000`
    
- 局域网访问（平板/手机）：确保设备连接同一 Wi-Fi

##  项目亮点与特色

1. **解决真实痛点**：专门针对“厨师不识字”设计，采用大图片+颜色标签交互，后厨数量徽章突出显示。
    
2. **灵活适配**：支持“手动加菜”，完美适配每日食材不固定的“明档”场景。
    
3. **数据可视化**：管理后台自带简易 BI 看板（近7日趋势、热销排行），辅助经营决策。
    
4. **低成本部署**：基于 JSON 文件存储，无需购买数据库服务器，甚至可部署在旧安卓手机上。
